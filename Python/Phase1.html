Interpreter vs Script Mode

Before writing any real code you need to understand where your code runs and why that matters. Most beginners skip this. Most beginners also spend 30 minutes confused about why their "program" disappeared the moment they closed a window.

REPL Interpreter / Interactive Mode

You type one line → Python executes it immediately → you see the result. Great for testing ideas, checking values, or just messing around. Python gives you a >> prompt. Run it by typing python or python3 in your terminal.

# In your terminal: python3
# Then you type line by line:

>>> 2 + 3
5
>>> print("hello")
hello
>>> x = 10
>>> x * 5
50
Key thing: In interactive mode, just typing an expression (like 2 + 3) shows the result automatically. In a script, it won't — you'd need print(). This trips people up constantly.
FILE Script Mode

You write all your code in a .py file, then run the whole file at once. This is how real programs work. Python reads it top to bottom, executes everything, then stops.

# my_program.py — run with: python3 my_program.py

name = "Alice"
age = 25

print(f"Name: {name}")
print(f"Age: {age}")

# Output when you run it:
# Name: Alice
# Age: 25
Note: In script mode, expressions don't auto-print. 2 + 3 on its own line does absolutely nothing visible. You must use print() explicitly.
When to use which?
SituationUse
Experimenting / testingInterpreter

Indentation Rules

Most languages use curly braces {} to group code blocks. Python uses whitespace instead. This isn't a quirky preference — it's enforced by the language. Get it wrong and your code won't run.

RULE The Golden Rule

Use 4 spaces per indentation level. Most editors do this automatically when you press Tab. The only actual rule is: be consistent. You can technically use 2 spaces or even tabs, but you absolutely cannot mix them.

# CORRECT — 4 spaces each level
if True:
    print("level 1")      # 4 spaces
    if True:
        print("level 2")  # 8 spaces
WRONG Common Mistakes
# ERROR: mixing tabs and spaces
if True:
    print("spaces here")    # 4 spaces
	print("tab here")       # TAB — CRASH

# ERROR: wrong amount of indentation
if True:
  print("2 spaces")       # fine, BUT
    print("4 spaces")     # INCONSISTENT — crash

# ERROR: missing indentation entirely
if True:
print("no indent")       # IndentationError
Real talk: Python will throw an IndentationError and refuse to run. It's not forgiving. Set your editor to "insert spaces on Tab" and this problem disappears entirely.
DEMO What Indentation Actually Does

Indentation tells Python which code belongs inside a block. This isn't just style — it defines the structure of your program.

score = 85

if score >= 50:
    print("inside the if — only runs if score >= 50")
    print("also inside the if")

print("outside the if — always runs")

# Output:
# inside the if — only runs if score >= 50
# also inside the if
# outside the if — always runs

Data Types

Everything in Python has a type. Python is "dynamically typed" — meaning you don't declare types explicitly, Python figures it out. That doesn't mean types don't matter. It means the errors you get when you confuse them are more confusing.

INT Integers — whole numbers
age = 25
score = -10
big = 1_000_000    # underscores for readability — totally valid

print(type(age))   # <class 'int'>
print(10 // 3)      # 3  ← floor division (drops remainder)
print(10 % 3)       # 1  ← modulo (remainder only)
print(2 ** 8)       # 256 ← exponentiation
FLOAT Floats — decimal numbers
price = 9.99
pi = 3.14159

print(0.1 + 0.2)   # 0.30000000000000004 ← welcome to floating point

# Convert between int and float:
print(int(9.9))    # 9   ← truncates, does NOT round
print(float(5))    # 5.0
0.1 + 0.2 ≠ 0.3 exactly. This is a floating-point precision issue that exists in basically every programming language. For money/finance, use the decimal module instead of floats.
COMPLEX Complex numbers — with imaginary parts

You'll rarely use these unless you're doing signal processing, maths, or physics. Python supports them natively using j (not i like in maths).

z = 3 + 4j
print(z.real)    # 3.0
print(z.imag)    # 4.0
print(abs(z))     # 5.0  ← magnitude
STR Strings — text

Text in Python is wrapped in single or double quotes — both work, pick one and be consistent. Strings are immutable — once created, you can't change individual characters.

name = "Alice"
greeting = 'Hello'

# f-strings — the modern way to embed variables
print(f"{greeting}, {name}!")   # Hello, Alice!

# Common string operations
print(name.upper())           # ALICE
print(name.lower())           # alice
print(len(name))              # 5
print(name[0])               # A  ← indexing starts at 0
print(name[-1])              # e  ← last character
print(name[1:3])             # li ← slicing

# Concatenation
print("Hello" + " World")    # Hello World
print("ha" * 3)               # hahaha
BOOL Booleans — True or False

Only two possible values: True or False (capital first letter — it matters). Booleans power all your logic and conditions.

is_raining = True
is_sunny = False

# Comparison operators produce booleans
print(5 > 3)      # True
print(5 == 5)     # True  ← == not =
print(5 != 3)     # True
print(5 >= 10)    # False

# Logical operators
print(True and False)  # False
print(True or False)   # True
print(not True)         # False

# "Truthy" and "Falsy" — Python treats these as False:
# 0, 0.0, "", None, [], {}
print(bool(0))    # False
print(bool(""))   # False
print(bool(42))   # True
= vs ==: = assigns a value. == checks equality. Confusing them is probably the single most common beginner mistake in all of programming.

Control Flow

Without control flow, your program just runs from top to bottom, does the same thing every time, and is basically useless. Control flow lets you make decisions and repeat actions.

IF if / elif / else

Check a condition. If it's true, do something. Otherwise, check another condition. Otherwise, do a default thing. The colon : at the end of each line is not optional.

score = 72

if score >= 90:
    print("Grade: A")
elif score >= 70:
    print("Grade: B")   # ← this runs
elif score >= 50:
    print("Grade: C")
else:
    print("Grade: F")

# Output: Grade: B
Python checks conditions top to bottom and stops at the first true one. So order matters. Put your most specific conditions first.
WHILE while loops — repeat while true

Keeps running a block of code as long as a condition is true. If the condition never becomes false, you have an infinite loop. Python will not save you from this.

count = 0

while count < 5:
    print(f"count is {count}")
    count += 1     # ESSENTIAL — without this, infinite loop

# Output:
# count is 0
# count is 1
# count is 2
# count is 3
# count is 4

# break — exit loop immediately
n = 0
while True:      # deliberately infinite
    if n == 3:
        break    # exits the loop
    n += 1
FOR for loops — iterate over a sequence

The most common loop in Python. For every item in a collection, do something. With range() you can loop a specific number of times.

# Loop over a list
fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(fruit)

# Loop a specific number of times with range()
for i in range(5):       # 0, 1, 2, 3, 4
    print(i)

for i in range(1, 6):    # 1, 2, 3, 4, 5
    print(i)

for i in range(0, 10, 2):  # 0, 2, 4, 6, 8 (step=2)
    print(i)

# continue — skip current iteration, carry on
for i in range(5):
    if i == 2:
        continue   # skip when i is 2
    print(i)
# Output: 0 1 3 4  (2 is missing)
range() explained: range(stop) → 0 to stop-1. range(start, stop) → start to stop-1. range(start, stop, step) → start to stop-1 in increments of step. The end is always exclusive — this confuses everyone at first.
COMBO Putting it all together
# FizzBuzz — the classic example
for i in range(1, 16):
    if i % 15 == 0:
        print("FizzBuzz")
    elif i % 3 == 0:
        print("Fizz")
    elif i % 5 == 0:
        print("Buzz")
    else:
        print(i)

# Output: 1, 2, Fizz, 4, Buzz, Fizz, 7, 8, Fizz, Buzz, 11, Fizz, 13, 14, FizzBuzz
